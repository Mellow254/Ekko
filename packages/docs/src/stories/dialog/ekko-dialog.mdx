<Subtitle>A modal surface for confirmations, sign-in flows, and brief detail views.</Subtitle>

`ekko-dialog` wraps the native `<dialog>` element. It inherits all the platform behaviour ŌĆö
top-layer rendering, `::backdrop`, focus trap (when modal), `Escape` to close, return value
support ŌĆö and adds a styled close button, slot-based footer, header (heading + description),
loading state, alert variant, and configurable backdrop click-to-close.

---

## When to use

Use `ekko-dialog` for:

- Confirmations ŌĆö Delete, Discard changes, Sign out
- Sign-in / authentication flows that interrupt the current task
- Brief detail views that need user attention before they continue

For non-blocking notifications ŌĆö toasts, inline banners ŌĆö use a toast/banner pattern (not in
Ekko). For navigation overlays attached to a screen edge, consider a drawer (not in Ekko).

## Properties & attributes

| Attribute | Property | Type | Default | Description |
| --- | --- | --- | --- | --- |
| `open` | `open` | `boolean` | `false` | Open state. Reflects the inner dialog's open state ŌĆö toggling the attribute opens/closes the dialog. |
| `modal` | `modal` | `boolean` | `false` | Controls which native method is used. `showModal()` when set (top layer + backdrop + focus trap), `show()` when not. |
| `size` | `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Width preset. Padding and typography scale with the size. |
| `disabled` | `disabled` | `boolean` | `false` | Disables interaction. Dims the dialog and blocks pointer events. |
| `loading` | `loading` | `boolean` | `false` | Shows a centered spinner, dims body and footer, and sets `aria-busy="true"` on the inner dialog. |
| `hide-close` | `hideClose` | `boolean` | `false` | Hides the built-in close button (and removes the right-padding it reserves on the header). |
| `close-on-backdrop` | `closeOnBackdrop` | `boolean` | `false` | When set, clicking the backdrop closes a modal dialog. Off by default ŌĆö most a11y guidance recommends explicit dismissal. |
| `heading` | `heading` | `string` | `''` | Heading text. When set, the inner dialog gets `aria-labelledby` pointing at the heading. |
| `description` | `description` | `string` | `''` | Supporting description rendered below the heading. Merged into `aria-describedby`. |
| `close-label` | ŌĆö | `string` | `'Close'` | Accessible name for the built-in close button. |
| `alert` | ŌĆö | `boolean` | `false` | Renders the inner dialog with `role="alertdialog"`. Use for destructive confirmations and error announcements. |
| `aria-label` | ŌĆö | `string` | ŌĆö | Accessible name. Forwarded to the inner `<dialog>`. Takes precedence over `aria-labelledby` derived from `heading`. |
| `aria-describedby` | ŌĆö | `string` | ŌĆö | Describing element id(s). Merged with the internal description id when `description` is set. |

Plus the `returnValue` getter/setter ŌĆö proxies the native dialog's `returnValue`. Read after
close, write before close, or pass to `close(value)`.

## Methods

| Method | Returns | Description |
| --- | --- | --- |
| `show()` | `void` | Opens as a non-modal. Stays in document flow; no backdrop, no focus trap. Sets `modal = false`. |
| `showModal()` | `void` | Opens as a modal. Renders in the top layer, paints the `::backdrop`, traps focus, and closes on `Escape`. Sets `modal = true`. |
| `close(returnValue?)` | `void` | Closes the dialog and stores the optional `returnValue`. The value is readable on the host's `returnValue` property and on the `ekko-close` event detail. |

Both `show()` and `showModal()` no-op when `disabled`. Calling `showModal()` on a dialog that
is already open as a non-modal is a no-op ŌĆö close it first.

## Events

| Event | `detail` shape | When fired |
| --- | --- | --- |
| `ekko-open` | `{ modal: boolean }` | After the dialog opens via `show()` or `showModal()`. The `modal` flag tells you which method was used. |
| `ekko-close` | `{ returnValue: string }` | After the dialog closes (close button, `close()`, form submit with `method="dialog"`, or the native `close` event). |
| `ekko-cancel` | `{ originalEvent: Event }` | When the user presses `Escape` (modal only). **Cancelable** ŌĆö calling `event.preventDefault()` keeps the dialog open. |
| `close` | native `Event` | Native close event from the inner `<dialog>`. Bubbles out of Shadow DOM. |
| `cancel` | native `Event` | Native cancel event from the inner `<dialog>`. Bubbles out of Shadow DOM. |

All Ekko events `bubble` and `compose` (cross the Shadow boundary).

ekko-dialog::part(base) {
  border: none;
}
ekko-dialog::part(footer) {
  justify-content: space-between;
}

## Accessibility

### Keyboard

| Key | Action |
| --- | --- |
| `Tab` / `Shift+Tab` | Cycles focus inside the dialog. The native focus trap kicks in for modal dialogs (top-layer behaviour). |
| `Escape` | Fires `cancel` (and `ekko-cancel`) and closes a modal dialog. Call `event.preventDefault()` on `ekko-cancel` to keep it open. |
| `Enter` | Submits a `<form method="dialog">` if one wraps the focused control ŌĆö see [Form integration](#form-integration) below. |

The close button is part of the natural tab order inside the dialog.

### Screen reader announcements

| State | Announcement |
| --- | --- |
| Default | "&#123;heading&#125;, dialog" (when `heading` is set) or "&#123;aria-label&#125;, dialog" |
| Alert variant | "&#123;heading&#125;, alert dialog" ŌĆö interrupts immediately and requires user action |
| Loading | "&#123;heading&#125;, dialog, busy" |

`aria-labelledby` points at the inner heading id when `heading` is set, otherwise the host's
`aria-label` is forwarded to the inner dialog. `aria-describedby` is set to the description
element's id and merged with any external `aria-describedby` on the host.

### Focus management

When opened as a modal, the browser moves focus to the dialog automatically (the first
focusable child, or the dialog itself). On close, focus returns to the previously focused
element ŌĆö this is native `<dialog>` behaviour and works for both `show()` and `showModal()`.

### `role="alertdialog"` implications

Setting `alert` switches the inner role to `alertdialog`. Alert dialogs interrupt screen
readers immediately and require user action ŌĆö use sparingly for critical confirmations
("Discard unsaved changes?", "Permanently delete this project?"), not for routine flows.

## Form integration

Native `<dialog>` supports the `method="dialog"` form submission pattern: a form inside the
dialog with `method="dialog"` closes the dialog when submitted, and the submitter button's
`value` becomes the dialog's `returnValue`. No JavaScript required.

```html
<ekko-dialog id="confirm" heading="Discard changes?" description="You will lose unsaved edits.">
  <form method="dialog">
    <ekko-button slot="footer" type="submit" name="action" value="cancel" variant="ghost">
      Cancel
    </ekko-button>
    <ekko-button slot="footer" type="submit" name="action" value="discard" variant="danger">
      Discard
    </ekko-button>
  </form>
</ekko-dialog>

<script>
  document.getElementById('confirm').addEventListener('ekko-close', (e) => {
    console.log('User chose:', e.detail.returnValue); // "cancel" or "discard"
  });
</script>
```

You can also call `close(value)` directly from a click handler if you don't want a form
wrapper ŌĆö the result is the same.

## Things to know

### `:host { display: contents }` ŌĆö the host renders nothing

The inner `<dialog>` does the layout. CSS that targets the host element (e.g. `width`,
`margin`, `padding` on `ekko-dialog`) has no visible effect. Style via tokens or the
`base` part instead.

### The close button is always rendered

Use `hide-close` to hide it (and remove the right-padding it reserves on the header). The
button is in the tab order whenever it's visible, giving keyboard users a predictable exit.

### `close-on-backdrop` is opt-in

By default, clicking the dimmed backdrop does nothing ŌĆö this matches the most common a11y
best practice that modal dismissal should be intentional. Set `close-on-backdrop` only when
the dialog represents a recoverable, low-stakes action.

### Loading dims body and footer, not the close button

Setting `loading` opacity-dims the body and footer, shows the centered spinner, and sets
`aria-busy="true"` on the inner dialog. The close button stays interactive so users can
escape if a long-running action stalls.

### `ekko-cancel` is cancelable

The native dialog's `cancel` event fires on `Escape`. Ekko wraps it in `ekko-cancel`
(cancelable). Calling `event.preventDefault()` on either keeps the dialog open ŌĆö useful for
"are you sure?" confirmation patterns.

### `returnValue` persists across show/hide cycles

`returnValue` is exposed both as a property (read/write on the host) and via `close(value)`.
It's preserved across show/hide cycles until you reset it manually ŌĆö make sure to clear
stale values before reopening if you want clean state.

### `z-index` defaults to 200

Far above typical UI layers but below toasts. Override
`--ekko-dialog-shared-z-index` if you stack overlays differently in your app.
