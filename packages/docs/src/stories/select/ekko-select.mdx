<Title />
<Subtitle>A dropdown for picking one (or more) values from a known list of options.</Subtitle>

`ekko-select` wraps a native `<select>` inside Shadow DOM. Authors place `<option>` and
`<optgroup>` elements as light-DOM children ‚Äî the component clones them into the inner
select via a `MutationObserver`, so updates to a child's `value`, `disabled`, or label
propagate automatically. In browsers that support `appearance: base-select` (modern
Chromium), the component opts in to the customizable-select picker so the open list
renders with Ekko's tokens; everywhere else it falls back to the platform-native
dropdown without losing functionality.

---

## When to use

- Picking exactly one value from a known list of more than four options
- Picking multiple values from a known list (with the `multiple` attribute)
- Forms where the value space is fixed and finite (countries, plans, sizes, etc.)

For free-text input with suggestions, use a combobox (not currently in Ekko). For two
to four mutually exclusive options, prefer [`ekko-radio`](/?path=/docs/components-radio--docs)
so all choices stay visible at once. For an open multi-select with chips and search,
reach for a tag input rather than `multiple` on a select.

## Properties & attributes

| Attribute | Property | Type | Default | Description |
| --- | --- | --- | --- | --- |
| `name` | `name` | `string` | `''` | Form submission name. Forwarded to the inner `<select>`. |
| `value` | `value` | `string` | `''` | Initial value as an attribute; current value as a property. Setting the property updates the inner select and the form value. |
| `autocomplete` | ‚Äî | `string` | ‚Äî | Forwarded to the inner `<select>`. Use the standard tokens (`off`, `country`, etc.). |
| `visible-rows` | `visibleRows` | `number` | `0` | When greater than `1`, becomes the `size` attribute on the inner select ‚Äî switches it to listbox mode (an open scrolling list). |
| `size` | `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Ekko sizing scale (height, padding, font-size). Independent of `visible-rows`. |
| `multiple` | `multiple` | `boolean` | `false` | Allow multi-select. Forces listbox mode and removes the chevron / custom trigger. |
| `disabled` | `disabled` | `boolean` | `false` | Disables interaction. Sets `aria-disabled="true"` on the inner select and excludes the field from `FormData`. |
| `required` | `required` | `boolean` | `false` | Marks the field as required. Sets `aria-required="true"` and renders an asterisk next to the label. |
| `invalid` | `invalid` | `boolean` | `false` | Visual error state. Sets `aria-invalid="true"` and reveals the `error` message when present. |
| `label` | `label` | `string` | `''` | Visible label, associated to the inner select via `for`/`id`. |
| `placeholder` | `placeholder` | `string` | `''` | When set on a single-select with no value, prepends a hidden disabled `<option>` selected by default. Ignored when `multiple`. |
| `help` | ‚Äî | `string` | ‚Äî | Hint text rendered below the control and wired via `aria-describedby`. |
| `error` | ‚Äî | `string` | ‚Äî | Error message rendered (with `role="alert"`) when `invalid` is set. Replaces help text. |
| `full-width` | ‚Äî | `boolean` | `false` | Stretches the host to fill its container. |
| `aria-label` | ‚Äî | `string` | ‚Äî | Accessible name when no visible label is set. Forwarded to the inner `<select>`. |
| `aria-describedby` | ‚Äî | `string` | ‚Äî | Describing element id(s). Merged with the internal help/error ids and forwarded to the inner `<select>`. |

The component also exposes the inner select's collections and form metadata as
read-only getters:

| Property | Type | Description |
| --- | --- | --- |
| `options` | `HTMLOptionsCollection` | Live collection of all `<option>` elements inside the inner select. |
| `selectedOptions` | `HTMLCollectionOf<HTMLOptionElement>` | Live collection of selected options. The right way to read selections in `multiple` mode. |
| `selectedIndex` | `number` (read/write) | Index of the first selected option. Settable to change the selection programmatically. |
| `form` | `HTMLFormElement \| null` | The host form, via `ElementInternals`. |
| `validity` | `ValidityState` | Forwarded from the inner select. |
| `validationMessage` | `string` | Forwarded from the inner select. |

## Events

| Event | `detail` shape | When fired |
| --- | --- | --- |
| `ekko-input` | `{ value: string, originalEvent: Event }` | The inner select dispatched `input` (the value changed during interaction). |
| `ekko-change` | `{ value: string, originalEvent: Event }` | The inner select dispatched `change` (the user committed a selection). |
| `input` | native `Event` | Standard DOM input. Bubbles out of Shadow DOM. |
| `change` | native `Event` | Standard DOM change. Bubbles out of Shadow DOM. |

Both `ekko-input` and `ekko-change` set `bubbles: true` and `composed: true`. In
`multiple` mode, `detail.value` reports the value of the most recently changed option
(matching native `<select>` behaviour) ‚Äî read `selectedOptions` for the full set.

## Accessibility

### Keyboard (closed picker)

| Key | Action |
| --- | --- |
| `Tab` / `Shift+Tab` | Move focus to/from the select |
| `Space` / `Enter` / `Alt+Down` | Open the picker |
| `Down` / `Up` | Move selection to the next / previous option (with picker closed in some platforms) |
| `Home` / `End` | Jump to the first / last option |
| Letter keys | Type-ahead ‚Äî jumps to the next option whose label starts with the typed character |

### Keyboard (open picker)

| Key | Action |
| --- | --- |
| `Down` / `Up` | Move highlight |
| `Home` / `End` | Highlight first / last option |
| `Enter` | Commit the highlighted option and close |
| `Esc` | Close without changing the value |

### Multiple-select mode

| Input | Action |
| --- | --- |
| `Click` on an option | Select only that option |
| `Shift+Click` | Select the contiguous range from the last selection to the click |
| `Ctrl+Click` (Windows/Linux) / `Cmd+Click` (macOS) | Toggle individual options without affecting others |
| `Shift+Down` / `Shift+Up` | Extend the selection by one |

### ARIA wiring

`disabled`, `invalid`, and `required` map automatically:

- `disabled` ‚Üí native `disabled` + `aria-disabled="true"` on the inner select
- `invalid` ‚Üí `aria-invalid="true"` on the inner select
- `required` ‚Üí native `required` + `aria-required="true"` on the inner select, plus a
  visible asterisk next to the label

The internal `<label>` is associated to the inner `<select>` via `for`/`id`, so clicking
the label focuses the control. `aria-describedby` is composed: any external value the
author set on the host is preserved and merged with the internal help id (when help is
present and the field is valid) or the internal error id (when `invalid` and `error`
are present).

When no `label` is provided, set `aria-label` on the host ‚Äî it is forwarded to the inner
select.

## Form integration

```html
<form>
  <ekko-select name="size" label="Size" required placeholder="Pick one">
    <option value="sm">Small</option>
    <option value="md">Medium</option>
    <option value="lg">Large</option>
  </ekko-select>
  <ekko-button type="submit">Continue</ekko-button>
</form>
```

`FormData` includes `name=value` for the current selection. In `multiple` mode each
selected option contributes its own entry ‚Äî `formData.getAll('toppings')` returns the
array.

`form.reset()` restores the value to whatever the `value` attribute held at construction
time, via `formResetCallback`. `<fieldset disabled>` propagates through
`formDisabledCallback` and disables the select with no extra wiring.

### Placeholder pattern

When `placeholder` is set on a single-select and no `value` is selected, the component
prepends a hidden, disabled `<option value="">` and selects it by default. This is the
common UX pattern of "Choose one‚Ä¶" ‚Äî the placeholder cannot be re-selected once the user
moves to a real option, and it is excluded from the picker dropdown. The placeholder is
ignored in `multiple` mode (where there is no concept of an unselected list).

See [Forms (FACE)](/?path=/docs/foundations-forms-face--docs) for the full form integration story.

## Things to know

### Options must be light-DOM children

Place `<option>` and `<optgroup>` directly inside `<ekko-select>`. The component clones
them into the shadow tree, so JavaScript event listeners or DOM-attached state on the
**originals** does not reach the **clones**. Set `value`, `disabled`, and labels
declaratively, and let the `MutationObserver` re-sync when an attribute changes.

### `value` attribute vs. `value` property

The `value` attribute is the **initial** selection. The `value` property is the
**current** selection. Setting the property writes through to the inner select and
updates the form value; setting the attribute also reseeds the value during the next
sync pass and is what `form.reset()` restores.

### `visible-rows > 1` switches to listbox mode

When `visible-rows="2"` (or higher) is set, the inner select gets a `size` attribute,
the chevron is hidden, and the custom `<button>` trigger is removed. The control renders
as an open scrolling list of N rows. `multiple` always implies listbox mode regardless
of `visible-rows`.

### Browser support for `appearance: base-select`

Modern Chromium browsers support [the customizable-select spec](https://developer.mozilla.org/en-US/docs/Web/CSS/appearance#values),
which is what unlocks tokenized option styling, hover/checked states inside the picker,
and rich `<option>` content via `<selectedcontent>`. The component detects support with
`CSS.supports('appearance', 'base-select')` and only injects the trigger button +
`data-base-picker` attribute when supported. In other browsers the component still works
‚Äî the platform native dropdown is used and all the closed-state tokens still apply.

### MutationObserver scope

The observer watches `childList`, `subtree`, and `attributes`. Adding, removing, or
attribute-mutating any descendant triggers a re-sync of the cloned options. There is a
`#syncing` re-entry guard so the observer's own writes into the shadow tree do not
trigger it recursively.

### `selectedIndex` is read/write

Use the property to programmatically move the selection without touching `value` ‚Äî
useful when option labels are duplicated. In `multiple` mode it returns the index of the
first selected option; for the full set, use `selectedOptions`.
